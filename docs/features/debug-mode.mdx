## 概述

TUI (REPL) 模式需要真實終端，無法直接通過 VS Code launch 啓動調試。使用 **attach 模式**連接到正在執行的 Bun 進程。

## 步驟

### 1. 終端啓動 inspect 服務

```bash
bun run dev:inspect
```

會輸出類似 `ws://localhost:8888/xxxxxxxx` 的地址。

### 2. VS Code 附着調試器

1. 在 `src/` 檔案中打斷點
2. F5 → 選擇 **"Attach to Bun (TUI debug)"**

> **注意**：`dev:inspect` 和 `launch.json` 中的 WebSocket 地址會在每次啓動時變化，需要同步更新兩處。

## 原理

`dev:inspect` 腳本實際執行的是：

```bash
bun --inspect-wait=localhost:8888/<token> run scripts/dev.ts
```

Bun 的 `--inspect-wait` 參數啓動一個 Chrome DevTools Protocol 相容的 inspect 服務，等待調試器連接後纔開始執行。VS Code 的 `bun` 擴展通過 WebSocket 連接到這個地址實現 attach。

## JetBrains IDE

理論上 JetBrains 系列（WebStorm / IntelliJ 等）也支援 attach 到 Bun inspect 服務（Run → Attach to Process），但尚未實際驗證過。如果你驗證成功，歡迎補充文件。

## 相關文件

| 文件 | 說明 |
|---|---|
| `package.json` → `dev:inspect` | 啓動 inspect 服務的 npm script |
| `.vscode/launch.json` | VS Code attach 調試設定 |
| `scripts/dev.ts` | dev 模式入口，注入 MACRO defines |
