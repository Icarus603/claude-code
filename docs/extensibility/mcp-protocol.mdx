## 架構總覽：從設定到可用工具

```
settings.json: { mcpServers: { "my-db": { command: "npx", args: [...] } } }
  ↓
getAllMcpConfigs()                    ← 合併 user/project/local 三級設定
  ↓
useManageMCPConnections()             ← React Hook 管理連接生命週期
  ↓
connectToServer(name, config)         ← memoize 快取（lodash memoize）
  ├── 建立 Transport（stdio/sse/http/...）
  ├── new Client()                    ← @modelcontextprotocol/sdk
  ├── client.connect(transport)       ← 超時控制（MCP_TIMEOUT, 預設 30s）
  └── 返回 MCPServerConnection        ← { connected | failed | needs-auth | pending }
  ↓
fetchToolsForClient(client)           ← LRU(20) 快取
  ├── client.request({ method: 'tools/list' })
  └── 每個工具包裝爲 MCPTool            ← 統一 Tool 介面
  ↓
assembleToolPool()                    ← 合併內置工具 + MCP 工具
  ↓
工具名格式: mcp__<serverName>__<toolName>  ← buildMcpToolName()
```

## 7 種傳輸層實現

`connectToServer()`（`client.ts:596-1643`）根據 `config.type` 分發到不同的 Transport 實現：

| 傳輸類型 | Transport 類 | 適用場景 | 認證方式 |
|----------|-------------|---------|---------|
| `stdio`（預設） | `StdioClientTransport` | 本地子進程 | 無 |
| `sse` | `SSEClientTransport` | 遠程 SSE 服務 | `ClaudeAuthProvider` + OAuth |
| `http` | `StreamableHTTPClientTransport` | HTTP 流 | `ClaudeAuthProvider` + OAuth |
| `sse-ide` | `SSEClientTransport` | IDE 集成 | lockfile token |
| `ws-ide` | `WebSocketTransport` | IDE WebSocket | `X-Claude-Code-Ide-Authorization` |
| `ws` | `WebSocketTransport` | WebSocket 服務 | session ingress token |
| `claudeai-proxy` | `StreamableHTTPClientTransport` | claude.ai 代理 | OAuth bearer + 401 重試 |

### stdio 傳輸的進程管理

stdio 類型的 MCP 服務器作爲子進程執行，cleanup 時採用 **信號升級策略**（`client.ts:1431-1564`）：

```
SIGINT (100ms) → SIGTERM (400ms) → SIGKILL
```

總清理時間上限 600ms，防止 MCP 服務器關閉阻塞 CLI 退出。

### 遠程傳輸的認證狀態機

SSE/HTTP 類型使用 `ClaudeAuthProvider` 實現 OAuth 認證流程。認證失敗時進入 `needs-auth` 狀態，並寫入 15 分鐘 TTL 的快取文件（`mcp-needs-auth-cache.json`），避免重複彈出認證提示。

```
連接嘗試 → 401 Unauthorized
  ↓
handleRemoteAuthFailure()
  ├── logEvent('tengu_mcp_server_needs_auth')
  ├── setMcpAuthCacheEntry(name)         ← 寫入 15min TTL 快取
  └── return { type: 'needs-auth' }      ← UI 顯示認證提示
```

## 連接快取與重連機制

`connectToServer` 使用 lodash `memoize` 快取連接對象，快取 key 爲 `${name}-${JSON.stringify(config)}`。

### 快取失效觸發

當連接關閉時（`client.onclose`），清除所有相關快取（`client.ts:1376-1404`）：

```typescript
client.onclose = () => {
  const key = getServerCacheKey(name, serverRef)
  fetchToolsForClient.cache.delete(name)      // 工具快取
  fetchResourcesForClient.cache.delete(name)  // 資源快取
  fetchCommandsForClient.cache.delete(name)   // 命令快取
  connectToServer.cache.delete(key)           // 連接快取
}
```

### 連接降級檢測

遠程傳輸有 **連續錯誤計數器**（`client.ts:1229`）：

```typescript
let consecutiveConnectionErrors = 0
const MAX_ERRORS_BEFORE_RECONNECT = 3
```

遇到終端錯誤（ECONNRESET、ETIMEDOUT、EPIPE 等）連續 3 次後，主動關閉 transport 觸發重連。對於 HTTP 傳輸，還檢測 session 過期（404 + JSON-RPC code -32001）。

### 請求級超時保護

每個 HTTP 請求使用獨立的 `setTimeout` 超時（`wrapFetchWithTimeout`，`client.ts:493`），而非共享 `AbortSignal.timeout()`。原因是 Bun 對 AbortSignal.timeout 的 GC 是惰性的——每個請求約 2.4KB 原生內存，即使請求毫秒級完成也要等 60s 纔回收。

```typescript
const controller = new AbortController()
const timer = setTimeout(c => c.abort(...), MCP_REQUEST_TIMEOUT_MS, controller)
timer.unref?.()  // 不阻止進程退出
```

## 工具發現：從 MCP 到 Tool 介面

`fetchToolsForClient()`（`client.ts:1745-2000`）使用 `memoizeWithLRU` 快取（上限 20），將 MCP 工具轉換爲 Claude Code 的統一 Tool 介面：

```typescript
const fullyQualifiedName = buildMcpToolName(client.name, tool.name)
// 結果: "mcp__my-db__query"
```

### 工具描述截斷

MCP 工具描述上限 2048 字符（`MAX_MCP_DESCRIPTION_LENGTH`）。OpenAPI 生成的 MCP 服務器曾觀察到 15-60KB 的描述文件。

### 工具能力標註

每個 MCP 工具根據 `tool.annotations` 自動標註：

| 註解 | 映射到 | 含義 |
|------|--------|------|
| `readOnlyHint` | `isReadOnly()` + `isConcurrencySafe()` | 只讀，可並行 |
| `destructiveHint` | `isDestructive()` | 破壞性操作 |
| `openWorldHint` | `isOpenWorld()` | 開放世界（不可枚舉） |
| `title` | `userFacingName()` | 顯示名稱 |

### MCP 工具的權限檢查

MCP 工具預設返回 `{ behavior: 'passthrough' }`（`client.ts:1816-1834`），意味着它們始終進入權限確認流程。工具名使用 `mcp__` 前綴精確匹配權限規則。

## MCP 工具的執行鏈路

```
AI 生成 tool_use: { name: "mcp__my-db__query", input: { sql: "..." } }
  ↓
MCPTool.call()                               ← client.ts:1835
  ├── ensureConnectedClient()                ← 確保連接有效（重連）
  ├── callMCPToolWithUrlElicitationRetry()   ← 帶 Elicitation 重試
  │   ├── client.request({ method: 'tools/call' })
  │   ├── 處理圖片結果（resize + persist）
  │   └── 內容截斷（mcpContentNeedsTruncation）
  ├── McpSessionExpiredError → 重試一次
  └── 返回 { data: content, mcpMeta }
```

### Session 過期自動重試

HTTP 傳輸的 MCP session 可能過期。檢測到 `McpSessionExpiredError` 後自動重試一次（`client.ts:1862`），因爲 `ensureConnectedClient()` 已經清除了快取並建立了新連接。

### 內容截斷與持久化

大型 MCP 工具輸出通過 `truncateMcpContentIfNeeded` 截斷，二進制內容（圖片）通過 `persistBinaryContent` 寫入文件並返回檔案路徑。圖片自動 resize（`maybeResizeAndDownsampleImageBuffer`）。

## MCP 連接的併發控制

```typescript
// 本地服務器併發連接數
getMcpServerConnectionBatchSize()    // 預設 3

// 遠程服務器併發連接數
getRemoteMcpServerConnectionBatchSize()  // 預設 20
```

本地 MCP 服務器（stdio）是重量級的子進程，預設限制 3 個併發連接。遠程服務器是輕量級 HTTP 請求，允許 20 個併發。

## 實際設定示例

```json
// settings.json 中的 MCP 設定
{
  "mcpServers": {
    "my-database": {
      "command": "npx",
      "args": ["@my-org/db-mcp-server"],
      "env": { "DB_URL": "postgres://..." }
    },
    "remote-api": {
      "type": "http",
      "url": "https://api.example.com/mcp"
    }
  }
}
```

設定後，AI 的工具列表中會出現 `mcp__my-database__query` 和 `mcp__remote-api__*` 工具——與內置工具使用相同的權限檢查鏈路和 UI 渲染。
