## 概述

Claude Code 的 GrowthBook 系統支援通過環境變量連接自定義 GrowthBook 服務器，實現遠程 feature flag 控制。

- **有設定時**：連接你的 GrowthBook 實例，拉取並快取 feature 值
- **無設定時**：所有 feature 讀取直接返回程式碼中的預設值，零網絡請求

## 環境變量

| 變量 | 必填 | 說明 |
|---|---|---|
| `CLAUDE_GB_ADAPTER_URL` | 是 | GrowthBook API 地址，如 `https://gb.example.com/` |
| `CLAUDE_GB_ADAPTER_KEY` | 是 | GrowthBook SDK Client Key，如 `sdk-xxxxx` |

兩個變量都設置時啓用適配器模式，否則完全跳過 GrowthBook。

## 使用方式

### 基本用法

```bash
CLAUDE_GB_ADAPTER_URL=https://gb.example.com/ \
CLAUDE_GB_ADAPTER_KEY=sdk-abc123 \
bun run dev
```

### 不使用 GrowthBook（預設行爲）

```bash
bun run dev
# 所有 getFeatureValue_CACHED_MAY_BE_STALE("xxx", defaultValue) 直接返回 defaultValue
```

## GrowthBook 服務端設定

### 步驟

1. **部署 GrowthBook 服務端**（Docker 自託管或 Cloud 版）
2. **建立 Environment**（如 `production`）
3. **建立 SDK Connection**，獲得 SDK Key（即 `CLAUDE_GB_ADAPTER_KEY`）
4. **按需添加 Feature**，key 和類型見下方列表

### 核心原則

- **不設定任何 feature 也能正常執行**——程式碼中每個呼叫都提供了預設值
- 只建立你想遠程控制的 feature，其餘走程式碼預設
- GrowthBook 上配了某個 feature 後，其值會覆蓋程式碼中的預設值

## 讀取優先級鏈

每個 feature 的值按以下順序解析，第一個命中即返回：

```
1. CLAUDE_INTERNAL_FC_OVERRIDES 環境變量（JSON 對象覆蓋）
   ↓ 未命中
2. growthBookOverrides 設定（~/.claude.json，僅 ant 構建）
   ↓ 未命中
3. 內存快取（remoteEvalFeatureValues，本次進程從服務器拉取）
   ↓ 未命中
4. 磁盤快取（~/.claude.json 的 cachedGrowthBookFeatures）
   ↓ 未命中
5. 程式碼中的 defaultValue 參數
```

## 快取與刷新機制

| 機制 | 說明 |
|---|---|
| **磁盤快取** | `~/.claude.json` 的 `cachedGrowthBookFeatures` 字段，跨進程持久化 |
| **週期刷新** | 每 6 小時自動從服務器拉取最新值（`setInterval` + `unref`） |
| **初始化超時** | 首次連接超時 5 秒，超時後使用磁盤快取或預設值 |
| **Auth 變更** | 登錄/登出時自動銷燬並重建客戶端 |

## 實現細節

修改了 2 個文件共 3 處：

1. **`src/constants/keys.ts`** — `getGrowthBookClientKey()` 優先讀取 `CLAUDE_GB_ADAPTER_KEY`
2. **`src/services/analytics/growthbook.ts`** — `isGrowthBookEnabled()` 適配器模式下直接啓用
3. **`src/services/analytics/growthbook.ts`** — base URL 優先使用 `CLAUDE_GB_ADAPTER_URL`

所有 130+ 個呼叫方文件無需修改。
