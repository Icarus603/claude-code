## 這份白皮書是什麼

這是對 Anthropic 官方發佈的 **Claude Code CLI** 的**逆向工程分析**。

源碼經過反編譯處理（TypeScript 單檔 bundle 逆向），保留了核心功能模組，但包含大量 `unknown`/`never`/`{}` 類型錯誤——這些不影響 Bun 執行時執行，但意味着我們的分析基於執行時行爲 + 殘留源碼結構，而非原始源碼。

**這不是：**
- 官方文件或使用教程
- API 參考手冊
- Claude Code 的功能推銷

**這是：**
- 一個生產級 agentic system 的架構解構
- 每個設計決策背後的"爲什麼"
- 可複用的工程模式：agentic loop、工具抽象、上下文工程、安全縱深防禦

## 逆向過程中最精妙的設計決策

### 1. Agentic Loop 的自愈能力

`src/query.ts` 實現的核心循環不是簡單的"發請求→收響應"。它是一個**自愈的狀態機**：

- API 返回錯誤（限流、token 超限）→ 自動重試/降級
- 工具執行超時 → 後臺化 + 通知機制
- 對話過長觸發 compaction → 壓縮歷史後無縫繼續
- 用戶中斷 → 生成 `UserInterruptionMessage` 讓 AI 理解發生了什麼

這不是"if-else 堆疊"，而是讓 AI 自己根據上下文決定下一步——即使發生了意外。

### 2. 上下文工程的分層策略

AI 沒有真正的"記憶"，Claude Code 通過精心分層營造了這個幻覺：

| 層 | 機制 | 持久性 |
|----|------|--------|
| **System Prompt** | 專案結構、git 狀態、CLAUDE.md | 每輪重建 |
| **對話歷史** | 完整的 User/Assistant/Tool 訊息 | 會話內 |
| **Compaction** | 自動壓縮過長對話爲摘要 | 壓縮後替代原始訊息 |
| **Memory 文件** | 跨會話持久化的筆記 | 永久（用戶可控） |
| **File History** | 文件修改時間戳快照 | 會話內 |

`src/context.ts` 組裝 System Prompt 時的策略是：**不變內容在前、變化內容在後**——這利用了 API 的快取機制，前綴不變時可以複用快取 token。

### 3. 工具系統的權限雙軌制

`src/tools/BashTool/shouldUseSandbox.ts` 展示了一個精巧的雙重安全模型：

- **應用層**：權限規則決定"能不能執行"（白名單/黑名單/用戶確認）
- **OS 層**：沙箱決定"執行時能做什麼"（檔案系統/網絡/進程隔離）

兩層的信任假設不同：應用層信任用戶設定，OS 層不信任任何東西。即使 AI 繞過了應用層權限（理論上不可能，但縱深防禦），OS 層沙箱仍然限制實際危害。

### 4. Feature Flag 的全局開關

`src/entrypoints/cli.tsx` 中一行程式碼決定了整個系統的行爲：

```typescript
const feature = (_name: string) => false;
```

所有 `feature('FLAG_NAME')` 呼叫返回 `false`——這意味着 Anthropic 內部的實驗功能（COORDINATOR_MODE、KAIROS、PROACTIVE 等）全部禁用。在官方構建中，這些 flag 通過 Bun 的 `bun:bundle` 在編譯時注入，不同用戶羣體看到不同功能。

這是一個**漸進式發佈架構**：同一個程式碼庫，通過 feature flag 控制功能可見性，而不需要維護多個分支。

### 5. Compaction 的分檔策略

`src/services/compact/` 實現了三種壓縮策略：

- **Micro-compact**：單次工具輸出過長時，截斷結果
- **Auto-compact**：對話 token 接近上限時，自動壓縮歷史
- **Reactive-compact**：API 返回 token 超限錯誤時，緊急壓縮後重試

這不是簡單的"砍掉舊訊息"——而是用 AI 自身來總結之前的對話，保留語義信息。壓縮後插入一條 `TombstoneMessage` 標記邊界。
