## 概述

Claude Code 支援通過 Sentry 捕獲執行時異常並上報到你自己指定的 Sentry 實例。

- **設定了 `SENTRY_DSN`**：自動初始化 Sentry SDK，捕獲未處理異常和關鍵錯誤
- **未設定**：所有 Sentry 呼叫均爲 no-op，零開銷

## 環境變量

| 變量 | 必填 | 說明 |
|---|---|---|
| `SENTRY_DSN` | 是 | Sentry 專案 DSN，如 `https://xxx@o123456.ingest.sentry.io/789` |

只需要這一個變量，設置後即啓用。

## 使用方式

### 自託管 Sentry

```bash
SENTRY_DSN=https://public_key@your-sentry.example.com/123 \
bun run dev
```

### Sentry Cloud (SaaS)

```bash
SENTRY_DSN=https://public_key@o123456.ingest.sentry.io/789 \
bun run dev
```

### 不使用 Sentry（預設行爲）

```bash
bun run dev
# SENTRY_DSN 未設置，所有 sentry 函數爲 no-op
```

## Sentry 服務端設定

### 步驟

1. **部署 Sentry 實例**（Docker 自託管 或 使用 [sentry.io](https://sentry.io) Cloud）
2. **建立 Project**，選擇 **Node.js** 平臺
3. 取得專案的 **DSN**（Settings → Projects → Client Keys → DSN）
4. 將 DSN 設置爲 `SENTRY_DSN` 環境變量

## 功能詳情

### 錯誤捕獲

- **自動捕獲**：`SentryErrorBoundary` 包裹關鍵 React 組件，捕獲渲染錯誤
- **手動上報**：`errorLogSink` 在寫入錯誤日誌時同步上報到 Sentry
- **優雅關閉**：進程退出時 `closeSentry()` 確保事件發送完畢（2s 超時）

### 安全過濾

`beforeSend` 鉤子會自動剝離以下敏感 header：

- `authorization`
- `x-api-key`
- `cookie`
- `set-cookie`

### 忽略的錯誤類型

以下錯誤模式會被忽略，不會上報：

| 錯誤 | 原因 |
|---|---|
| `ECONNREFUSED` / `ECONNRESET` / `ENOTFOUND` / `ETIMEDOUT` | 網絡不可達，不可操作 |
| `AbortError` / `The user aborted a request` | 用戶主動取消 |
| `CancelError` | 交互式取消信號 |

### 其他設定

- **採樣率**：`sampleRate: 1.0`（捕獲全部錯誤事件）
- **麪包屑上限**：`maxBreadcrumbs: 20`（控制 payload 體積）
- **性能事務**：已關閉（`beforeSendTransaction` 返回 `null`），僅上報錯誤

## API

| 函數 | 說明 |
|---|---|
| `initSentry()` | 初始化 SDK，在 `src/entrypoints/init.ts` 中自動呼叫 |
| `captureException(error, context?)` | 手動上報異常，可附加額外上下文 |
| `setTag(key, value)` | 設置標籤，用於 Sentry 面板分組過濾 |
| `setUser({ id, email, username })` | 設置用戶上下文，用於錯誤歸因 |
| `closeSentry(timeoutMs?)` | 刷出隊列並關閉客戶端，進程退出時呼叫 |
| `isSentryInitialized()` | 檢查是否已初始化 |

## 實現文件

| 文件 | 說明 |
|---|---|
| `src/utils/sentry.ts` | 核心 SDK 初始化與封裝 |
| `src/components/SentryErrorBoundary.ts` | React Error Boundary 組件 |
| `src/utils/errorLogSink.ts` | 錯誤日誌 sink，集成 `captureException` |
| `src/utils/gracefulShutdown.ts` | 優雅退出，呼叫 `closeSentry()` |
| `src/entrypoints/init.ts` | 啓動時呼叫 `initSentry()` |
