## 執行鏈路總覽

一條 `Agent(prompt="修復 bug")` 呼叫的完整路徑：

```
AI 生成 tool_use: { prompt: "修復 bug", subagent_type: "Explore" }
  ↓
AgentTool.call()                              ← 入口（AgentTool.tsx:239）
  ├── 解析 effectiveType（fork vs 命名 agent）
  ├── filterDeniedAgents()                    ← 權限過濾
  ├── 檢查 requiredMcpServers                 ← MCP 依賴驗證（最長等 30s）
  ├── assembleToolPool(workerPermissionContext) ← 獨立組裝工具池
  ├── createAgentWorktree()                   ← 可選 worktree 隔離
  ↓
runAgent()                                    ← 核心執行（runAgent.ts:248）
  ├── getAgentSystemPrompt()                  ← 構建 agent 專屬 system prompt
  ├── initializeAgentMcpServers()             ← agent 級 MCP 服務器
  ├── executeSubagentStartHooks()             ← Hook 注入
  ├── query()                                 ← 進入標準 agentic loop
  │   ├── 訊息流逐條 yield
  │   └── recordSidechainTranscript()         ← JSONL 持久化
  ↓
finalizeAgentTool()                           ← 結果彙總
  ├── 提取文本內容 + usage 統計
  └── mapToolResultToToolResultBlockParam()   ← 格式化爲 tool_result
```

## 兩種子 Agent 路徑：命名 Agent vs Fork

`AgentTool.call()` 根據是否提供 `subagent_type` 走兩條完全不同的路徑（`AgentTool.tsx:322-356`）：

| 維度 | 命名 Agent（`subagent_type` 指定） | Fork 子進程（`subagent_type` 省略） |
|------|-------------------------------------|--------------------------------------|
| **觸發條件** | `subagent_type` 有值 | `isForkSubagentEnabled()` && 未指定類型 |
| **System Prompt** | Agent 自身的 `getSystemPrompt()` | 繼承父 Agent 的完整 System Prompt |
| **工具池** | `assembleToolPool()` 獨立組裝 | 父 Agent 的原始工具池（`useExactTools: true`） |
| **上下文** | 僅任務描述 | 父 Agent 的完整對話歷史（`forkContextMessages`） |
| **模型** | 可獨立指定 | 繼承父模型（`model: 'inherit'`） |
| **權限模式** | Agent 定義的 `permissionMode` | `'bubble'`（上浮到父終端） |
| **目的** | 專業任務委派 | Prompt Cache 命中率優化 |

Fork 路徑的設計核心是 **Prompt Cache 共享**：所有 fork 子進程共享父 Agent 的完整 `assistant` 訊息（所有 `tool_use` 塊），用相同的佔位符 `tool_result` 填充，只有最後一個 `text` 塊包含各自的指令。這使得 API 請求前綴字節完全一致，最大化快取命中。

```typescript
// forkSubagent.ts:142 — 所有 fork 子進程的佔位結果
const FORK_PLACEHOLDER_RESULT = 'Fork started — processing in background'

// buildForkedMessages() 構建：
// [assistant(全量 tool_use), user(placeholder_results..., 子進程指令)]
```

### Fork 遞歸防護

Fork 子進程保留 Agent 工具（爲了 cache-identical tool defs），但通過兩道防線防止遞歸 fork（`AgentTool.tsx:332`）：

1. **`querySource` 檢查**（壓縮安全）：`context.options.querySource === 'agent:builtin:fork'`
2. **訊息掃描**（降級兜底）：檢測 `<fork-boilerplate>` 標籤

## 工具池的獨立組裝

子 Agent 不繼承父 Agent 的工具限制——它的工具池完全獨立組裝（`AgentTool.tsx:573-577`）：

```typescript
const workerPermissionContext = {
  ...appState.toolPermissionContext,
  mode: selectedAgent.permissionMode ?? 'acceptEdits'
}
const workerTools = assembleToolPool(workerPermissionContext, appState.mcp.tools)
```

關鍵設計決策：
- **權限模式獨立**：子 Agent 使用 `selectedAgent.permissionMode`（預設 `acceptEdits`），不受父 Agent 當前模式的限制
- **MCP 工具繼承**：`appState.mcp.tools` 包含所有已連接的 MCP 工具，子 Agent 自動獲得
- **Agent 級 MCP 服務器**：`runAgent()` 中的 `initializeAgentMcpServers()` 可以爲特定 Agent 額外連接專屬 MCP 服務器

### 工具過濾的 resolveAgentTools

`runAgent.ts:500-502` 在工具組裝後進一步過濾：

```typescript
const resolvedTools = useExactTools
  ? availableTools                           // Fork: 直接使用父工具
  : resolveAgentTools(agentDefinition, availableTools, isAsync).resolvedTools
```

`resolveAgentTools()` 會根據 Agent 定義中的 `tools` 字段過濾可用工具，將 `['*']` 映射爲全量工具。

## Worktree 隔離機制

`isolation: "worktree"` 參數讓子 Agent 在獨立的 git worktree 中工作（`AgentTool.tsx:590-593`）：

```typescript
const slug = `agent-${earlyAgentId.slice(0, 8)}`
worktreeInfo = await createAgentWorktree(slug)
```

Worktree 生命週期：
1. **建立**：在 `.git/worktrees/` 下建立獨立工作副本
2. **CWD 覆蓋**：`runWithCwdOverride(worktreePath, fn)` 讓所有檔案操作在 worktree 中執行
3. **路徑翻譯**：Fork + worktree 時注入路徑翻譯通知（`buildWorktreeNotice`）
4. **清理**（`cleanupWorktreeIfNeeded`）：
   - Hook-based worktree → 始終保留
   - 有變更 → 保留，返回 `worktreePath`
   - 無變更 → 自動刪除

## 生命週期管理：同步 vs 異步

### 異步 Agent（後臺執行）

當 `run_in_background=true` 或 `selectedAgent.background=true` 時，Agent 立即返回 `async_launched` 狀態（`AgentTool.tsx:686-764`）：

```
registerAsyncAgent(agentId, ...)      ← 註冊到 AppState.tasks
  ↓ (void — 火後不管)
runAsyncAgentLifecycle()              ← 後臺執行
  ├── runAgent().onCacheSafeParams    ← 進度摘要初始化
  ├── 訊息流迭代
  ├── completeAsyncAgent()            ← 標記完成
  ├── classifyHandoffIfNeeded()       ← 安全檢查
  └── enqueueAgentNotification()      ← 通知主 Agent
```

異步 Agent 獲得獨立的 `AbortController`，不與父 Agent 共享——用戶按 ESC 取消主執行緒不會殺掉後臺 Agent。

### 同步 Agent（前臺執行）

同步 Agent 的關鍵特性是 **可後臺化**（`AgentTool.tsx:818-833`）：

```typescript
const registration = registerAgentForeground({
  autoBackgroundMs: getAutoBackgroundMs() || undefined  // 預設 120s
})
backgroundPromise = registration.backgroundSignal.then(...)
```

在 agentic loop 的每次迭代中，系統用 `Promise.race` 競爭下一條訊息和後臺化信號：

```typescript
const raceResult = await Promise.race([
  nextMessagePromise.then(r => ({ type: 'message', result: r })),
  backgroundPromise  // 超過 autoBackgroundMs 觸發
])
```

後臺化後，前臺迭代器被終止（`agentIterator.return()`），新的 `runAgent()` 以 `isAsync: true` 重新啓動，當前臺的輸出文件繼續寫入。

## 結果回傳格式

`mapToolResultToToolResultBlockParam()` 根據狀態返回不同格式（`AgentTool.tsx:1298-1375`）：

| 狀態 | 返回內容 |
|------|---------|
| `completed` | 內容 + `<usage>` 塊（token/tool_calls/duration） |
| `async_launched` | agentId + outputFile 路徑 + 操作指引 |
| `teammate_spawned` | agent_id + name + team_name |
| `remote_launched` | taskId + sessionUrl + outputFile |

對於一次性內置 Agent（Explore、Plan），`<usage>` 塊被省略——每週節省約 1-2 Gtok 的上下文窗口。

## MCP 依賴的等待機制

如果 Agent 聲明瞭 `requiredMcpServers`，`call()` 會等待這些服務器連接完成（`AgentTool.tsx:371-410`）：

```typescript
const MAX_WAIT_MS = 30_000     // 最長等 30 秒
const POLL_INTERVAL_MS = 500   // 每 500ms 輪詢
```

早期退出條件：任何必需服務器進入 `failed` 狀態時立即停止等待。工具可用性通過 `mcp__` 前綴工具名解析（`mcp__serverName__toolName`）判斷。
